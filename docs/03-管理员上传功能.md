# 管理员上传功能文档

## 概述

管理员上传功能允许管理员通过 Bot 手动创建媒体合集，上传图片/视频，设置合集信息和访问权限，最终生成可分享的深链接。

## 权限要求

- 用户角色必须是 `admin` 或 `super_admin`
- 普通用户和 VIP 用户无法使用此功能

## 功能流程

### 完整上传流程

```
管理员发送 /upload 命令
  ↓
Bot: "请发送媒体文件（图片/视频），完成后发送 /done"
  ↓
管理员: 发送多个图片/视频
  ↓
Bot: 临时存储到内存/Redis（记录 file_id）
  ↓
管理员: /done
  ↓
Bot: "已接收 15 个媒体文件"
Bot: "请输入合集名称"
  ↓
管理员: "可爱猫咪合集"
  ↓
Bot: "请输入合集描述（可选，发送 /skip 跳过）"
  ↓
管理员: "各种可爱的猫咪图片和视频"
  ↓
Bot: "请输入标签（用空格分隔，可选，发送 /skip 跳过）"
  ↓
管理员: "猫咪 可爱 宠物"
  ↓
Bot: "请选择访问权限" [公开] [VIP]
  ↓
管理员: 点击 [VIP]
  ↓
Bot: 保存到数据库，生成深链接
  ↓
Bot: "✅ 创建成功！
      📦 合集名称: 可爱猫咪合集
      🔗 深链接: t.me/bot?start=abc123
      📊 媒体数量: 15
      🔒 访问权限: VIP"
```

## 状态机设计

使用 aiogram 的 FSM (Finite State Machine) 管理上传流程。

### 状态定义

```python
from aiogram.fsm.state import State, StatesGroup

class UploadStates(StatesGroup):
    waiting_for_media = State()      # 等待上传媒体
    waiting_for_name = State()       # 等待输入名称
    waiting_for_description = State() # 等待输入描述
    waiting_for_tags = State()       # 等待输入标签
    waiting_for_permission = State()  # 等待选择权限
```

### 状态转换图

```
[开始] → waiting_for_media
         ↓ (/done)
       waiting_for_name
         ↓
       waiting_for_description
         ↓ (/skip 或输入描述)
       waiting_for_tags
         ↓ (/skip 或输入标签)
       waiting_for_permission
         ↓ (选择权限)
       [完成]
```

## 详细功能说明

### 1. 启动上传 (/upload)

**命令**: `/upload`

**权限检查**:
```python
async def check_admin_permission(user_id: int) -> bool:
    user = await get_user(user_id)
    return user.role in ["admin", "super_admin"]
```

**响应**:
```
📤 开始创建新合集

请发送媒体文件（图片或视频）
• 可以一次发送多个文件
• 支持媒体组（最多10个）
• 完成后发送 /done

💡 提示: 发送 /cancel 可随时取消
```

**状态**: 进入 `waiting_for_media`

### 2. 接收媒体文件

**支持的媒体类型**:
- 图片 (photo)
- 视频 (video)
- 文档 (document) - 可选

**处理逻辑**:
```python
async def handle_media_upload(message: Message, state: FSMContext):
    """接收并临时存储媒体"""
    # 获取当前状态数据
    data = await state.get_data()
    media_list = data.get("media_list", [])

    # 提取 file_id
    if message.photo:
        file_id = message.photo[-1].file_id
        file_type = "photo"
    elif message.video:
        file_id = message.video.file_id
        file_type = "video"
    else:
        await message.answer("❌ 不支持的文件类型")
        return

    # 添加到列表
    media_list.append({
        "file_id": file_id,
        "file_type": file_type,
        "file_unique_id": message.photo[-1].file_unique_id if message.photo else message.video.file_unique_id,
        "caption": message.caption
    })

    # 更新状态
    await state.update_data(media_list=media_list)

    # 提示
    await message.answer(f"✅ 已接收 {len(media_list)} 个文件")
```

**媒体组处理**:
```python
# Telegram 会将媒体组拆分成多条消息
# 需要使用 media_group_id 识别同一组
async def handle_media_group(messages: list[Message], state: FSMContext):
    """批量处理媒体组"""
    for message in messages:
        await handle_media_upload(message, state)
```

**限制**:
- 单个合集不限制媒体数量（可配置）
- 单个文件大小不限制 （Bot API 获取文件id）

### 3. 完成媒体上传 (/done)

**命令**: `/done`

**验证**:
```python
async def validate_media_list(media_list: list) -> tuple[bool, str]:
    """验证媒体列表"""
    if not media_list:
        return False, "❌ 还没有上传任何媒体文件"

    if len(media_list) > 1000:
        return False, "❌ 媒体数量超过限制（最多1000个）"

    return True, ""
```

**响应**:
```
✅ 已接收 15 个媒体文件

📝 请输入合集名称
（建议简短明了，如：可爱猫咪合集）
```

**状态**: 进入 `waiting_for_name`

### 4. 输入合集名称

**验证规则**:
- 长度: 1-100 字符
- 不能包含特殊字符: `< > / \ | : * ? "`
- 不能与现有合集重名（可选）

**处理逻辑**:
```python
async def handle_collection_name(message: Message, state: FSMContext):
    """处理合集名称"""
    name = message.text.strip()

    # 验证
    if not name or len(name) > 100:
        await message.answer("❌ 名称长度必须在 1-100 字符之间")
        return

    if any(char in name for char in '<>/\\|:*?"'):
        await message.answer("❌ 名称不能包含特殊字符")
        return

    # 保存
    await state.update_data(name=name)

    # 下一步
    await message.answer(
        "📝 请输入合集描述（可选）\n\n"
        "发送 /skip 跳过此步骤"
    )
    await state.set_state(UploadStates.waiting_for_description)
```

### 5. 输入合集描述

**可选字段**: 用户可以发送 `/skip` 跳过

**验证规则**:
- 长度: 0-500 字符

**处理逻辑**:
```python
async def handle_collection_description(message: Message, state: FSMContext):
    """处理合集描述"""
    if message.text == "/skip":
        description = ""
    else:
        description = message.text.strip()
        if len(description) > 500:
            await message.answer("❌ 描述长度不能超过 500 字符")
            return

    await state.update_data(description=description)

    await message.answer(
        "🏷️ 请输入标签（用空格分隔，可选）\n\n"
        "示例: 猫咪 可爱 宠物\n"
        "发送 /skip 跳过此步骤"
    )
    await state.set_state(UploadStates.waiting_for_tags)
```

### 6. 输入标签

**可选字段**: 用户可以发送 `/skip` 跳过

**格式**: 用空格分隔，如 `猫咪 可爱 宠物`

**验证规则**:
- 最多 10 个标签
- 每个标签长度: 1-20 字符

**处理逻辑**:
```python
async def handle_collection_tags(message: Message, state: FSMContext):
    """处理标签"""
    if message.text == "/skip":
        tags = []
    else:
        tags = message.text.strip().split()

        # 验证
        if len(tags) > 10:
            await message.answer("❌ 标签数量不能超过 10 个")
            return

        for tag in tags:
            if len(tag) > 20:
                await message.answer(f"❌ 标签 '{tag}' 长度超过 20 字符")
                return

    await state.update_data(tags=tags)

    # 显示权限选择按钮
    keyboard = InlineKeyboardMarkup(inline_keyboard=[
        [
            InlineKeyboardButton(text="🌍 公开", callback_data="access_public"),
            InlineKeyboardButton(text="💎 VIP", callback_data="access_vip")
        ]
    ])

    await message.answer(
        "🔒 请选择访问权限:",
        reply_markup=keyboard
    )
    await state.set_state(UploadStates.waiting_for_permission)
```

### 7. 选择访问权限

**选项**:
- 🌍 公开: 所有用户可访问
- 💎 VIP: 仅 VIP 用户可访问

**处理逻辑**:
```python
async def handle_access_permission(callback: CallbackQuery, state: FSMContext):
    """处理权限选择"""
    access_level = "public" if callback.data == "access_public" else "vip"

    await state.update_data(access_level=access_level)

    # 获取所有数据
    data = await state.get_data()

    # 创建合集
    collection = await create_collection(
        name=data["name"],
        description=data.get("description", ""),
        tags=data.get("tags", []),
        access_level=access_level,
        media_list=data["media_list"],
        created_by=callback.from_user.id
    )

    # 清除状态
    await state.clear()

    # 发送成功消息
    access_emoji = "🌍" if access_level == "public" else "💎"
    await callback.message.answer(
        f"✅ 创建成功！\n\n"
        f"📦 合集名称: {collection.name}\n"
        f"🔗 深链接: t.me/{BOT_USERNAME}?start={collection.deep_link_code}\n"
        f"📊 媒体数量: {len(data['media_list'])}\n"
        f"🔒 访问权限: {access_emoji} {access_level.upper()}"
    )
```

## 数据库操作

### 创建合集

```python
async def create_collection(
    name: str,
    description: str,
    tags: list[str],
    access_level: str,
    media_list: list[dict],
    created_by: int
) -> Collection:
    """创建合集并保存媒体"""

    # 生成深链接码
    deep_link_code = generate_deep_link_code()

    # 创建合集
    collection = Collection(
        name=name,
        description=description,
        tags=tags,
        deep_link_code=deep_link_code,
        access_level=access_level,
        media_count=len(media_list),
        created_by=created_by,
        created_at=datetime.now()
    )

    db.add(collection)
    await db.flush()  # 获取 collection.id

    # 批量插入媒体
    media_objects = []
    for index, media_data in enumerate(media_list):
        media = Media(
            collection_id=collection.id,
            file_id=media_data["file_id"],
            file_unique_id=media_data["file_unique_id"],
            file_type=media_data["file_type"],
            caption=media_data.get("caption"),
            order_index=index
        )
        media_objects.append(media)

    db.add_all(media_objects)
    await db.commit()

    return collection
```

### 深链接生成

```python
import secrets
import string

def generate_deep_link_code(length: int = 8) -> str:
    """生成唯一的深链接码"""
    chars = string.ascii_letters + string.digits
    while True:
        code = ''.join(secrets.choice(chars) for _ in range(length))
        # 检查是否已存在
        if not await collection_exists_by_code(code):
            return code
```

## 取消上传 (/cancel)

**命令**: `/cancel`

**功能**: 在任何步骤都可以取消上传流程

**处理逻辑**:
```python
async def cancel_upload(message: Message, state: FSMContext):
    """取消上传"""
    current_state = await state.get_state()

    if current_state is None:
        await message.answer("❌ 没有正在进行的上传任务")
        return

    await state.clear()
    await message.answer("✅ 已取消上传")
```

## 编辑现有合集

### 添加媒体到现有合集

**命令**: `/add_media {deep_link_code}`

**流程**:
```
/add_media abc123
  ↓
Bot: "请发送要添加的媒体文件"
  ↓
管理员: 发送媒体
  ↓
管理员: /done
  ↓
Bot: "✅ 已添加 5 个媒体到合集"
```

**处理逻辑**:
```python
async def add_media_to_collection(
    collection_id: int,
    media_list: list[dict]
) -> int:
    """添加媒体到现有合集"""

    # 获取当前最大 order_index
    max_order = await db.execute(
        select(func.max(Media.order_index))
        .where(Media.collection_id == collection_id)
    )
    start_index = (max_order.scalar() or -1) + 1

    # 插入新媒体
    media_objects = []
    for index, media_data in enumerate(media_list):
        media = Media(
            collection_id=collection_id,
            file_id=media_data["file_id"],
            file_unique_id=media_data["file_unique_id"],
            file_type=media_data["file_type"],
            caption=media_data.get("caption"),
            order_index=start_index + index
        )
        media_objects.append(media)

    db.add_all(media_objects)

    # 更新合集媒体数量
    await db.execute(
        update(Collection)
        .where(Collection.id == collection_id)
        .values(
            media_count=Collection.media_count + len(media_list),
            updated_at=datetime.now()
        )
    )

    await db.commit()
    return len(media_list)
```

## 错误处理

### 常见错误

1. **权限不足**: 非管理员尝试使用管理命令
2. **合集不存在**: 编辑/删除不存在的合集
3. **文件过大**: 超过 20MB 限制
4. **重复文件**: 相同 file_unique_id 的文件

### 错误消息

## 性能优化

1. **批量插入**: 使用 `add_all()` 批量插入媒体
2. **异步处理**: 所有数据库操作使用异步
3. **状态存储**: 使用 Redis 存储 FSM 状态，避免内存泄漏
4. **文件去重**: 使用 `file_unique_id` 防止重复上传

## 用户体验优化

1. **实时反馈**: 每上传一个文件都给予提示
2. **进度显示**: 显示已上传文件数量
3. **可撤销**: 任何步骤都可以 `/cancel` 取消
4. **智能提示**: 提供示例和格式说明
