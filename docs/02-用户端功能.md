# 用户端功能文档

## 概述

用户端功能是普通用户通过 Telegram Bot 访问和浏览媒体合集的核心功能。用户可以通过深链接直接访问合集，浏览媒体内容，以及搜索感兴趣的合集。

## 功能列表

1. 深链接访问合集
2. 分页浏览媒体
3. 搜索合集
4. 欢迎消息
5. 用户注册与权限检查

## 1. 深链接访问合集

### 功能描述

用户通过深链接 `t.me/yourbot?start=collection_abc123` 可以直接打开指定的合集，无需手动搜索。

### 深链接格式

```
t.me/{bot_username}?start={deep_link_code}
```

**示例**:
- `t.me/blackholebot?start=abc123`
- `t.me/blackholebot?start=cute_cats_2024`

### 处理流程

```
用户点击深链接
  ↓
Bot 接收 /start 命令，参数为 deep_link_code
  ↓
解析 deep_link_code
  ↓
查询数据库获取合集信息
  ↓
检查合集是否存在
  ↓
检查用户权限（VIP 合集需要 VIP 权限）
  ↓
获取合集的前 10 个媒体
  ↓
发送媒体组 + 分页按钮
```

### 权限检查逻辑

```python
def check_collection_access(user_role: str, collection_access_level: str) -> bool:
    """
    检查用户是否有权限访问合集

    规则:
    - public 合集: 所有用户可访问
    - vip 合集: 仅 vip、admin、super_admin 可访问
    """
    if collection_access_level == "public":
        return True

    if collection_access_level == "vip":
        return user_role in ["vip", "admin", "super_admin"]

    return False
```

### 响应示例

**成功访问**:
```

📝 描述: 各种可爱的猫咪图片和视频

[发送 10 个媒体文件]

按钮:
[◀️ 上一页] [1/5] [下一页 ▶️]
```

**权限不足**:
```

💎 升级为 VIP 用户即可访问

```

**合集不存在**:
```

🔍 使用 /search 搜索其他合集
```

## 2. 分页浏览媒体

### 功能描述

由于 Telegram 媒体组限制为 10 个文件，需要实现分页功能让用户浏览完整合集。

### 分页逻辑

- 每页显示 10 个媒体文件
- 使用 InlineKeyboard 实现翻页
- 支持上一页、下一页、跳转到首页/末页

### 按钮布局

```
[◀️ 上一页] [页码 1/5] [下一页 ▶️]
[🏠 首页] [📋 合集信息] [🔍 搜索]
```

### 翻页处理流程

```
用户点击 "下一页" 按钮
  ↓
Bot 接收 callback_query
  ↓
解析 callback_data: collection_abc123_page_2
  ↓
查询数据库获取第 2 页媒体（offset=10, limit=10）
  ↓
删除旧消息
  ↓
发送新的媒体组 + 更新按钮
```

### Callback Data 格式

```
collection_{deep_link_code}_page_{page_number}
```

**示例**:
- `collection_abc123_page_1`
- `collection_abc123_page_2`
- `collection_abc123_info` (查看合集信息)

### 数据库查询

```sql
-- 获取指定页的媒体
SELECT * FROM media
WHERE collection_id = ?
ORDER BY order_index ASC
LIMIT 10 OFFSET ?;

-- 计算总页数
SELECT CEIL(COUNT(*)::float / 10) as total_pages
FROM media
WHERE collection_id = ?;
```

### 性能优化

- 使用 Redis 缓存当前用户的浏览状态
- 预加载下一页数据（可选）
- 使用 `file_id` 直接发送，避免重复上传

## 3. 搜索合集

### 功能描述

用户可以通过关键词搜索合集名称和标签，快速找到感兴趣的内容。

### 搜索命令

```
/search 关键词
```

**示例**:
- `/search 猫咪`
- `/search 风景`

### 搜索流程

```
用户发送 /search 猫咪
  ↓
Bot 解析关键词 "猫咪"
  ↓
在数据库中模糊匹配合集名称和标签
  ↓
过滤用户无权访问的合集（VIP 合集）
  ↓
返回搜索结果列表（最多 10 个）
  ↓
用户点击结果，跳转到合集
```

### 搜索算法

**匹配规则**:
1. 合集名称包含关键词（优先级高）
2. 标签包含关键词
3. 描述包含关键词（可选）

**数据库查询**:
```sql
SELECT c.* FROM collections c
WHERE (
    c.name ILIKE '%' || ? || '%'
    OR c.tags @> ?::jsonb
    OR c.description ILIKE '%' || ? || '%'
)
AND (
    c.access_level = 'public'
    OR (c.access_level = 'vip' AND ? IN ('vip', 'admin', 'super_admin'))
)
ORDER BY
    CASE
        WHEN c.name ILIKE '%' || ? || '%' THEN 1
        WHEN c.tags @> ?::jsonb THEN 2
        ELSE 3
    END,
    c.created_at DESC
LIMIT 10;
```

## 5. 用户注册与权限检查

### 自动注册

用户首次使用 Bot 时自动注册到数据库：

```python
async def register_user(telegram_id: int, username: str, first_name: str):
    """
    自动注册新用户
    默认角色: user
    """
    user = User(
        telegram_id=telegram_id,
        username=username,
        first_name=first_name,
        role="user",
        created_at=datetime.now(),
        last_active_at=datetime.now()
    )
    await db.add(user)
    await db.commit()
```

### 权限等级

| 角色 | 权限 |
|------|------|
| `user` | 访问公开合集 |
| `vip` | 访问公开合集 + VIP 合集 |
| `admin` | 所有用户权限 + 管理功能 |
| `super_admin` | 所有权限 |

### 封禁用户

管理员可以封禁用户，封禁后用户无法使用任何功能：

```
用户发送任何命令
  ↓
检查 is_banned 字段
  ↓
如果被封禁，返回提示消息
  ↓
否则正常处理
```

**封禁提示**:
```
🚫 您已被管理员封禁

如有疑问，请联系管理员
```

## 错误处理

### 常见错误

1. **合集不存在**: 提示用户合集已删除或不存在
2. **权限不足**: 提示升级 VIP
3. **网络错误**: 提示稍后重试
4. **Bot 限流**: 自动等待后重试

### 错误消息示例

```python
ERROR_MESSAGES = {
```

## 性能优化

### 1. Redis 缓存

缓存热门合集信息，减少数据库查询：

```python
# 缓存合集信息
await redis.hset(
    f"collection:{deep_link_code}",
    mapping={
        "name": collection.name,
        "description": collection.description,
        "access_level": collection.access_level,
        "media_count": collection.media_count
    }
)
await redis.expire(f"collection:{deep_link_code}", 3600)  # 1小时
```

### 2. 分页查询优化

使用索引优化分页查询：

```sql
CREATE INDEX idx_media_collection_order ON media(collection_id, order_index);
```

### 3. 媒体组发送优化

批量获取 `file_id`，一次性发送媒体组：

```python
media_group = [
    InputMediaPhoto(media=file_id) for file_id in file_ids[:10]
]
await bot.send_media_group(chat_id, media_group)
```

## 用户体验优化

1. **快速响应**: 使用 `typing` 状态提示用户正在处理
2. **友好提示**: 所有错误都有清晰的提示信息
3. **按钮交互**: 使用 InlineKeyboard 减少用户输入
4. **进度提示**: 加载大量媒体时显示进度

## 安全考虑

1. **防止滥用**: 限制搜索频率（每分钟最多 10 次）
2. **输入验证**: 过滤恶意输入和 SQL 注入
3. **权限检查**: 每次访问都验证用户权限
4. **日志记录**: 记录所有用户操作，便于审计
