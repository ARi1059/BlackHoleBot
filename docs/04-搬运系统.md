# 搬运系统文档

## 概述

搬运系统是 BlackHoleBot 的核心功能之一，允许管理员通过多个 Telegram 账号自动从频道/群组批量转发媒体文件到 Bot，经过审核后创建合集。系统具备智能限流控制、自动账号切换、任务队列管理等高级特性。

## 技术架构

### 核心组件

1. **Telethon 客户端管理器** - 管理多个 Telegram 用户账号
2. **任务队列系统** - 确保同时只有一个任务执行
3. **限流控制器** - 处理 API 限流和自动切换
4. **文件接收器** - Bot 接收转发的文件并提取 file_id
5. **Web 管理界面** - 创建和监控搬运任务

### 技术栈

- **Telethon** - Telegram MTProto 客户端（用于用户账号）
- **aiogram** - Telegram Bot API（用于 Bot）
- **Redis** - 临时存储 file_id 和任务状态
- **PostgreSQL** - 持久化存储任务和账号信息
- **asyncio** - 异步任务调度

## 完整工作流程

```
Web 后台创建搬运任务
  ↓
设置来源频道 + 过滤条件
  ↓
任务进入队列（确保同时只有一个任务）
  ↓
系统根据优先级选择可用 session 账号
  ↓
启动 Telethon 客户端，遍历频道历史消息
  ↓
符合条件的媒体转发给 Bot
  ↓
【限流控制】
  - 每转发 500 个文件 → 暂停 3 分钟
  - API 限流 → 切换下一个 session 账号
  - Bot API 限流 → session 停止转发，等待 Bot 恢复
  ↓
Bot 接收文件，提取 file_id 存入 Redis
  ↓
【Bot 限流恢复机制】
  - Bot 检测到限流结束
  - 等待已转发媒体的 file_id 全部获取完成
  - 发送通知："✅ 限流前文件已全部存储，继续搬运"
  - Session 账号收到通知后继续转发
  ↓
任务完成，Web 后台显示待审核状态
  ↓
管理员在 Web 中查看预览
  ↓
填写合集信息（名称、描述、权限）
  ↓
点击确认 → 存入数据库，清空 Redis，生成深链接
```

## Session 账号管理

### 账号配置

每个 session 账号需要配置：
- **phone_number**: 手机号
- **api_id**: Telegram API ID
- **api_hash**: Telegram API Hash
- **session_string**: 登录后的 session 字符串（加密存储）
- **priority**: 优先级（数字越小越优先）

### 获取 API 凭证

1. 访问 https://my.telegram.org
2. 登录 Telegram 账号
3. 进入 "API development tools"
4. 创建应用获取 `api_id` 和 `api_hash`

### Session 登录流程

**通过 Web 界面登录**:

```
Web 后台 → Session 管理 → 添加账号
  ↓
输入 phone_number, api_id, api_hash
  ↓
点击 "发送验证码"
  ↓
后端启动 Telethon 客户端，发送验证码到手机
  ↓
用户输入验证码
  ↓
（如果启用两步验证）输入密码
  ↓
登录成功，生成 session_string
  ↓
加密后存入数据库
```

**代码实现**:

```python
from telethon import TelegramClient
from telethon.sessions import StringSession

async def login_session_account(
    phone: str,
    api_id: int,
    api_hash: str,
    code: str = None,
    password: str = None
) -> str:
    """登录 Telegram 账号并返回 session_string"""

    client = TelegramClient(
        StringSession(),
        api_id,
        api_hash
    )

    await client.connect()

    if not code:
        # 发送验证码
        await client.send_code_request(phone)
        return "code_sent"

    # 使用验证码登录
    try:
        await client.sign_in(phone, code)
    except SessionPasswordNeededError:
        if not password:
            return "password_required"
        await client.sign_in(password=password)

    # 获取 session_string
    session_string = client.session.save()

    await client.disconnect()

    return session_string
```

### Session 加密存储

使用 Fernet 对称加密保护 session_string：

```python
from cryptography.fernet import Fernet
import os

# 从环境变量读取密钥
ENCRYPTION_KEY = os.getenv("SESSION_ENCRYPTION_KEY")
cipher = Fernet(ENCRYPTION_KEY.encode())

def encrypt_session(session_string: str) -> str:
    """加密 session 字符串"""
    return cipher.encrypt(session_string.encode()).decode()

def decrypt_session(encrypted_session: str) -> str:
    """解密 session 字符串"""
    return cipher.decrypt(encrypted_session.encode()).decode()
```

### Session 优先级选择

系统根据以下规则选择 session 账号：

1. **优先级排序**: `priority` 字段（数字越小越优先）
2. **可用性检查**: `is_active = True`
3. **冷却状态**: `cooldown_until` 为空或已过期
4. **转发计数**: 优先选择 `transfer_count` 较小的账号

```python
async def get_next_available_session() -> SessionAccount | None:
    """获取下一个可用的 session 账号"""
    now = datetime.now()

    result = await db.execute(
        select(SessionAccount)
        .where(
            SessionAccount.is_active == True,
            or_(
                SessionAccount.cooldown_until == None,
                SessionAccount.cooldown_until < now
            )
        )
        .order_by(
            SessionAccount.priority.asc(),
            SessionAccount.transfer_count.asc()
        )
        .limit(1)
    )

    return result.scalar_one_or_none()
```

## 任务队列管理

### 单任务执行保证

系统确保同时只有一个搬运任务在执行，避免资源冲突和限流问题。

```python
class TaskQueue:
    def __init__(self):
        self.current_task_id: int | None = None
        self.queue: asyncio.Queue = asyncio.Queue()
        self.is_processing = False

    async def add_task(self, task_id: int):
        """添加任务到队列"""
        await self.queue.put(task_id)

        # 如果没有任务在执行，立即开始
        if not self.is_processing:
            asyncio.create_task(self._process_queue())

    async def _process_queue(self):
        """处理队列中的任务"""
        self.is_processing = True

        while not self.queue.empty():
            task_id = await self.queue.get()
            self.current_task_id = task_id

            try:
                await execute_transfer_task(task_id)
            except Exception as e:
                await log_task_error(task_id, str(e))
            finally:
                self.queue.task_done()

        self.current_task_id = None
        self.is_processing = False

# 全局队列实例
task_queue = TaskQueue()
```

### 任务状态

| 状态 | 描述 |
|------|------|
| `pending` | 等待执行 |
| `running` | 执行中 |
| `paused` | 已暂停（手动或所有 session 冷却中） |
| `waiting_bot` | 等待 Bot API 恢复 |
| `completed` | 已完成 |
| `failed` | 失败 |

## 限流控制系统

### 三种限流场景

#### 1. Session 账号限流（500 文件/周期）

**规则**: 每个 session 账号转发 500 个文件后，暂停 3 分钟

**处理逻辑**:

```python
class SessionRateLimiter:
    TRANSFER_LIMIT = 500
    COOLDOWN_MINUTES = 3

    async def check_and_update(self, session_id: int) -> bool:
        """检查是否需要冷却"""
        session = await get_session(session_id)

        if session.transfer_count >= self.TRANSFER_LIMIT:
            # 设置冷却时间
            cooldown_until = datetime.now() + timedelta(
                minutes=self.COOLDOWN_MINUTES
            )
            await db.execute(
                update(SessionAccount)
                .where(SessionAccount.id == session_id)
                .values(
                    cooldown_until=cooldown_until,
                    transfer_count=0  # 重置计数
                )
            )
            await db.commit()

            # 记录日志
            await log_task_event(
                task_id=current_task_id,
                event_type="session_cooldown",
                message=f"Session {session_id} 达到 500 文件限制，冷却 3 分钟"
            )

            return True  # 需要冷却

        # 增加计数
        await db.execute(
            update(SessionAccount)
            .where(SessionAccount.id == session_id)
            .values(transfer_count=session.transfer_count + 1)
        )
        await db.commit()

        return False  # 继续转发
```

#### 2. API ID/Hash 限流

**触发条件**: Telethon 抛出 `FloodWaitError`

**处理逻辑**: 自动切换到下一个可用 session 账号

```python
from telethon.errors import FloodWaitError

async def handle_api_rate_limit(task_id: int, current_session_id: int):
    """处理 API 限流"""

    # 标记当前 session 进入冷却
    await db.execute(
        update(SessionAccount)
        .where(SessionAccount.id == current_session_id)
        .values(
            cooldown_until=datetime.now() + timedelta(minutes=10)
        )
    )
    await db.commit()

    # 记录日志
    await log_task_event(
        task_id=task_id,
        event_type="api_rate_limit",
        message=f"Session {current_session_id} 遇到 API 限流，切换账号"
    )

    # 选择下一个可用 session
    next_session = await get_next_available_session()

    if next_session:
        # 更新任务使用的 session
        await db.execute(
            update(TransferTask)
            .where(TransferTask.id == task_id)
            .values(session_account_id=next_session.id)
        )
        await db.commit()

        # 继续任务
        return next_session
    else:
        # 没有可用 session，暂停任务
        await db.execute(
            update(TransferTask)
            .where(TransferTask.id == task_id)
            .values(status="paused")
        )
        await db.commit()

        await log_task_event(
            task_id=task_id,
            event_type="task_paused",
            message="所有 session 都在冷却中，任务已暂停"
        )

        return None
```

#### 3. Bot API 限流

**触发条件**: aiogram 抛出 `TelegramRetryAfter` 异常

**处理逻辑**:
1. Session 停止转发
2. 等待已转发文件的 file_id 全部获取完成
3. 通知管理员
4. 等待限流结束后继续

```python
from aiogram.exceptions import TelegramRetryAfter

class BotRateLimiter:
    def __init__(self):
        self.is_limited = False
        self.pending_file_ids: set[int] = set()  # 等待获取 file_id 的消息

    async def handle_rate_limit(
        self,
        task_id: int,
        retry_after: int
    ):
        """处理 Bot API 限流"""
        self.is_limited = True

        # 更新任务状态
        await db.execute(
            update(TransferTask)
            .where(TransferTask.id == task_id)
            .values(status="waiting_bot")
        )
        await db.commit()

        # 停止 session 转发
        await stop_session_transfer(task_id)

        # 记录日志
        await log_task_event(
            task_id=task_id,
            event_type="bot_rate_limit",
            message=f"Bot API 限流，等待 {retry_after} 秒"
        )

        # 等待所有 pending 的 file_id 获取完成
        while self.pending_file_ids:
            await asyncio.sleep(0.5)

        # 通知管理员
        await notify_admin(
            task_id,
            f"✅ 限流前文件已全部存储\n"
            f"⏳ 等待 Bot API 恢复（{retry_after} 秒）"
        )

        # 等待限流时间
        await asyncio.sleep(retry_after)

        # 恢复
        await self.resume_after_limit(task_id)

    async def resume_after_limit(self, task_id: int):
        """限流恢复后继续"""
        self.is_limited = False

        # 更新任务状态
        await db.execute(
            update(TransferTask)
            .where(TransferTask.id == task_id)
            .values(status="running")
        )
        await db.commit()

        # 通知管理员
        await notify_admin(
            task_id,
            "✅ Bot API 已恢复，继续搬运"
        )

        # 恢复 session 转发
        await resume_session_transfer(task_id)

# 全局实例
bot_rate_limiter = BotRateLimiter()
```

## 文件转发与接收

### Telethon 转发文件

```python
async def transfer_media_from_channel(
    task_id: int,
    session: SessionAccount,
    source_chat_id: int,
    filters: dict
):
    """从频道转发媒体到 Bot"""

    # 解密 session
    session_string = decrypt_session(session.session_string)

    # 创建客户端
    client = TelegramClient(
        StringSession(session_string),
        session.api_id,
        session.api_hash
    )

    await client.connect()

    try:
        # 获取频道实体
        channel = await client.get_entity(source_chat_id)

        # 遍历消息
        async for message in client.iter_messages(
            channel,
            limit=None,
            reverse=False
        ):
            # 检查是否需要停止（Bot 限流）
            if bot_rate_limiter.is_limited:
                await asyncio.sleep(1)
                continue

            # 应用过滤条件
            if not await apply_filters(message, filters):
                continue

            # 检查 session 限流
            if await session_rate_limiter.check_and_update(session.id):
                # 切换 session
                next_session = await handle_api_rate_limit(task_id, session.id)
                if not next_session:
                    break
                # 递归调用，使用新 session
                await transfer_media_from_channel(
                    task_id,
                    next_session,
                    source_chat_id,
                    filters
                )
                return

            # 转发到 Bot
            try:
                await client.forward_messages(
                    entity=BOT_USERNAME,
                    messages=message.id,
                    from_peer=channel
                )

                # 更新进度
                await update_task_progress(task_id, increment=1)

            except FloodWaitError as e:
                # API 限流
                next_session = await handle_api_rate_limit(task_id, session.id)
                if not next_session:
                    break
                # 切换 session 后继续
                await transfer_media_from_channel(
                    task_id,
                    next_session,
                    source_chat_id,
                    filters
                )
                return

    finally:
        await client.disconnect()

    # 任务完成
    await complete_transfer_task(task_id)
```

### Bot 接收文件

```python
from aiogram import Router, F
from aiogram.types import Message

router = Router()

@router.message(F.forward_from_chat)
async def receive_transferred_media(message: Message):
    """接收转发的媒体文件"""

    # 检查是否来自搬运任务
    # （可以通过特殊标记或检查当前任务）
    task_id = await get_current_transfer_task()
    if not task_id:
        return

    # 标记为 pending
    bot_rate_limiter.pending_file_ids.add(message.message_id)

    try:
        # 提取 file_id
        if message.photo:
            file_id = message.photo[-1].file_id
            file_unique_id = message.photo[-1].file_unique_id
            file_type = "photo"
        elif message.video:
            file_id = message.video.file_id
            file_unique_id = message.video.file_unique_id
            file_type = "video"
        else:
            return

        # 存入 Redis
        redis_key = f"task:{task_id}:files"
        await redis.rpush(redis_key, json.dumps({
            "file_id": file_id,
            "file_unique_id": file_unique_id,
            "file_type": file_type,
            "caption": message.caption
        }))

        # 设置过期时间（24 小时）
        await redis.expire(redis_key, 86400)

    except TelegramRetryAfter as e:
        # Bot API 限流
        await bot_rate_limiter.handle_rate_limit(task_id, e.retry_after)

    finally:
        # 移除 pending 标记
        bot_rate_limiter.pending_file_ids.discard(message.message_id)
```

## 过滤条件

### 支持的过滤器

1. **媒体类型**: photo / video / all
2. **关键词**: 消息文本或 caption 包含关键词
3. **日期范围**: 指定时间段内的消息
4. **文件大小**: 限制文件大小范围（可选）

### 过滤逻辑

```python
async def apply_filters(message, filters: dict) -> bool:
    """应用过滤条件"""

    # 媒体类型过滤
    filter_type = filters.get("type", "all")
    if filter_type == "photo" and not message.photo:
        return False
    if filter_type == "video" and not message.video:
        return False

    # 关键词过滤
    keywords = filters.get("keywords", [])
    if keywords:
        text = message.text or message.caption or ""
        if not any(keyword in text for keyword in keywords):
            return False

    # 日期范围过滤
    date_from = filters.get("date_from")
    date_to = filters.get("date_to")
    if date_from and message.date < date_from:
        return False
    if date_to and message.date > date_to:
        return False

    # 文件大小过滤（可选）
    max_size = filters.get("max_file_size")
    if max_size:
        file_size = 0
        if message.photo:
            file_size = message.photo[-1].file_size
        elif message.video:
            file_size = message.video.file_size

        if file_size > max_size:
            return False

    return True
```

### 审核流程

```

  ↓

  ↓
管理员点击 "审核任务"
  ↓
显示预览（前 20 个文件缩略图）
  ↓

  - 合集名称
  - 描述
  - 标签
  - 访问权限
  ↓

  ↓
从 Redis 读取所有 file_id
  ↓

  ↓

  ↓
生成深链接
  ↓
显示成功消息
```

### 代码实现

```python
async def approve_transfer_task(
    task_id: int,
    collection_data: dict
) -> Collection:
    """审核并创建合集"""

    # 从 Redis 获取文件列表
    redis_key = f"task:{task_id}:files"
    file_data_list = await redis.lrange(redis_key, 0, -1)

    if not file_data_list:
        raise ValueError("没有找到文件数据")

    media_list = [json.loads(data) for data in file_data_list]

    # 创建合集
    collection = await create_collection(
        name=collection_data["name"],
        description=collection_data.get("description", ""),
        tags=collection_data.get("tags", []),
        access_level=collection_data["access_level"],
        media_list=media_list,
        created_by=collection_data["created_by"]
    )

    # 清空 Redis
    await redis.delete(redis_key)

    # 更新任务状态
    await db.execute(
        update(TransferTask)
        .where(TransferTask.id == task_id)
        .values(
            status="completed",
            completed_at=datetime.now()
        )
    )
    await db.commit()

    return collection
```

## 错误处理与恢复

### 常见错误

1. **网络错误**: 自动重试 3 次
2. **Session 失效**: 标记为不可用，通知管理员
3. **频道不存在**: 任务失败，记录错误
4. **权限不足**: 任务失败，提示需要加入频道

### 任务恢复

系统重启后自动恢复未完成的任务：

```python
async def recover_pending_tasks():
    """恢复未完成的任务"""
    tasks = await db.execute(
        select(TransferTask)
        .where(TransferTask.status.in_(["running", "paused"]))
    )

    for task in tasks.scalars():
        await task_queue.add_task(task.id)
```

## 性能优化

1. **批量转发**: 使用 `forward_messages` 批量转发
2. **异步处理**: 所有操作使用 asyncio
3. **Redis 缓存**: 临时文件存储使用 Redis
4. **连接复用**: Telethon 客户端保持连接

## 安全考虑

1. **Session 加密**: 使用 Fernet 加密存储
2. **权限验证**: 仅管理员可创建任务
3. **频道验证**: 检查频道是否存在和可访问
4. **文件去重**: 使用 `file_unique_id` 防止重复
