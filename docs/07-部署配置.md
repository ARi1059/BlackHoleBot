# 部署配置文档

## 概述

本文档详细说明 BlackHoleBot 的部署流程，包括服务器环境配置、依赖安装、服务配置、HTTPS 设置、监控和备份策略。

## 系统要求

### 硬件要求

- **CPU**: 2 核心以上
- **内存**: 4GB 以上（推荐 8GB）
- **存储**: 50GB 以上 SSD
- **网络**: 稳定的网络连接，支持访问 Telegram API

### 软件要求

- **操作系统**: Ubuntu 20.04 LTS / 22.04 LTS（推荐）
- **Python**: 3.10+
- **Node.js**: 18+ (用于前端构建)
- **PostgreSQL**: 14+
- **Redis**: 7+
- **Nginx**: 1.18+

```bash
cp .env.example .env

```

**环境变量配置**:
```bash
# Bot 配置
BOT_TOKEN=your_bot_token_here
```

```bash
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```


### 8. 构建前端

```bash
cd web/frontend

cd ../..
```

## 服务配置

### 1. Bot 服务

创建 systemd 服务文件：

```bash
sudo nano /etc/systemd/system/blackholebot-bot.service
```

**内容**:
```ini
[Unit]
Description=BlackHoleBot Telegram Bot
After=network.target postgresql.service redis-server.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/blackholebot
Environment="PATH=/var/www/blackholebot/venv/bin"
ExecStart=/var/www/blackholebot/venv/bin/python bot/main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### 2. Web API 服务

```bash
sudo nano /etc/systemd/system/blackholebot-web.service
```

**内容**:
```ini
[Unit]
Description=BlackHoleBot Web API
After=network.target postgresql.service redis-server.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/blackholebot
Environment="PATH=/var/www/blackholebot/venv/bin"
ExecStart=/var/www/blackholebot/venv/bin/uvicorn web.main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### 3. 启动服务

```bash
# 重载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start blackholebot-bot
sudo systemctl start blackholebot-web

# 设置开机自启
sudo systemctl enable blackholebot-bot
sudo systemctl enable blackholebot-web

# 查看状态
sudo systemctl status blackholebot-bot
sudo systemctl status blackholebot-web
```

## Nginx 配置

### 1. 创建 Nginx 配置文件

```bash
sudo nano /etc/nginx/sites-available/blackholebot
```

**内容**:
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    # SSL 证书
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # SSL 配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 日志
    access_log /var/log/nginx/blackholebot_access.log;
    error_log /var/log/nginx/blackholebot_error.log;

    # 前端静态文件
    location / {
        root /var/www/blackholebot/web/frontend/dist;
        try_files $uri $uri/ /index.html;

        # 缓存静态资源
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # API 代理
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

        # WebSocket 超时
        proxy_read_timeout 3600s;
        proxy_send_timeout 3600s;
    }

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "no-referrer-when-downgrade" always;

    # 文件上传大小限制
    client_max_body_size 50M;
}
```

### 2. 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/blackholebot /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

## 备份策略

### 1. 数据库备份脚本

```bash
sudo nano /usr/local/bin/backup-blackholebot.sh
```

**内容**:
```bash
#!/bin/bash

# 配置
BACKUP_DIR="/var/backups/blackholebot"
DB_NAME="blackholebot"
DB_USER="blackholebot_user"
RETENTION_DAYS=7

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份文件名
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="$BACKUP_DIR/blackholebot_$TIMESTAMP.sql.gz"

# 执行备份
PGPASSWORD="your_secure_password" pg_dump -U $DB_USER -h localhost $DB_NAME | gzip > $BACKUP_FILE

# 删除旧备份
find $BACKUP_DIR -name "blackholebot_*.sql.gz" -mtime +$RETENTION_DAYS -delete

echo "Backup completed: $BACKUP_FILE"
```

```bash
# 设置权限
sudo chmod +x /usr/local/bin/backup-blackholebot.sh

# 添加到 crontab（每天凌晨 2 点备份）
sudo crontab -e
```

添加：
```
0 2 * * * /usr/local/bin/backup-blackholebot.sh >> /var/log/blackholebot-backup.log 2>&1
```

### 2. 恢复数据库

```bash
# 解压并恢复
gunzip -c /var/backups/blackholebot/blackholebot_20240115_020000.sql.gz | \
    PGPASSWORD="your_secure_password" psql -U blackholebot_user -h localhost blackholebot
```

## 性能优化

### 1. PostgreSQL 优化

```bash
sudo nano /etc/postgresql/14/main/postgresql.conf
```

**推荐配置**:
```
# 内存配置（根据服务器内存调整）
shared_buffers = 1GB
effective_cache_size = 3GB
maintenance_work_mem = 256MB
work_mem = 16MB

# 连接配置
max_connections = 100

# 查询优化
random_page_cost = 1.1  # SSD
effective_io_concurrency = 200
```

```bash
sudo systemctl restart postgresql
```

### 2. Redis 优化

```bash
sudo nano /etc/redis/redis.conf
```

**推荐配置**:
```
# 内存限制
maxmemory 512mb
maxmemory-policy allkeys-lru

# 持久化
save 900 1
save 300 10
save 60 10000

appendonly yes
appendfsync everysec
```

```bash
sudo systemctl restart redis-server
```

## 故障排查

### 常见问题

**1. Bot 无法启动**
```bash
# 查看日志
sudo journalctl -u blackholebot-bot -n 50

# 检查配置
python3 -c "from config import settings; print(settings.BOT_TOKEN)"
```

**2. 数据库连接失败**
```bash
# 测试连接
psql -U blackholebot_user -h localhost -d blackholebot

# 检查 PostgreSQL 状态
sudo systemctl status postgresql
```

**3. Redis 连接失败**
```bash
# 测试连接
redis-cli ping

# 检查 Redis 状态
sudo systemctl status redis-server
```

**4. Nginx 502 错误**
```bash
# 检查 Web API 是否运行
sudo systemctl status blackholebot-web

# 检查端口占用
sudo netstat -tlnp | grep 8000
```
