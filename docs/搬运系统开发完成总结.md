## 核心功能特性

### 智能限流控制

1. **Session 账号限流**
   - 每个 Session 转发 500 个文件后自动冷却 3 分钟
   - 自动重置计数器
   - 记录日志

2. **API ID/Hash 限流**
   - 检测 `FloodWaitError`
   - 自动切换到下一个可用 Session
   - 设置冷却时间

3. **Bot API 限流**
   - 检测 `TelegramRetryAfter`
   - 暂停转发，等待 pending 文件完成
   - 自动恢复转发

### 任务队列管理

- 确保同时只有一个任务执行
- 异步队列处理
- 任务暂停/恢复支持

### Session 账号管理

- 多账号支持
- 优先级排序
- 自动选择可用账号
- 加密存储 Session 字符串

### 过滤功能

- 媒体类型过滤（photo/video/all）
- 关键词过滤
- 日期范围过滤

## 使用流程

### 1. 创建搬运任务

```
管理员: /create_transfer
Bot: 请输入来源频道 ID 或用户名
管理员: @channel_username
Bot: 请选择要搬运的媒体类型 [图片] [视频] [全部]
管理员: 点击 [全部]
Bot: 请输入过滤关键词（可选）
管理员: /skip
Bot: 请输入任务名称
管理员: 测试搬运任务
Bot: ✅ 搬运任务创建成功！任务已加入队列
```

### 2. 监控任务

```

Bot: 显示所有任务列表及状态

管理员: /task_info 1
Bot: 显示任务详细信息和最近日志
```

### 3. 审核任务

```

管理员: 各种可爱的猫咪
Bot: 请输入标签（可选）
管理员: 猫咪 可爱
Bot: 请选择访问权限 [公开] [VIP]
管理员: 点击 [公开]
Bot: ✅ 合集创建成功！
     🔗 深链接: t.me/bot?start=abc123
```

## 技术亮点

1. **异步架构**: 全程使用 asyncio，高性能处理
2. **智能限流**: 三层限流机制，确保稳定运行
3. **自动切换**: Session 账号自动切换，无需人工干预
4. **状态管理**: FSM 状态机管理复杂交互流程
5. **加密安全**: Fernet 加密保护 Session 数据
6. **Redis 缓存**: 临时文件存储，减轻数据库压力
7. **日志记录**: 完整的任务日志，便于追踪和调试

## 部署建议

1. **环境准备**
   ```bash
   # 安装依赖
   pip install -r requirements.txt

   # 配置环境变量
   cp .env.example .env
   # 编辑 .env 填写配置

   # 生成加密密钥
   python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
   ```

2. **数据库初始化**
   ```bash
   # 创建数据库
   createdb blackholebot

   # 运行迁移
   alembic upgrade head

   # 创建管理员
   python scripts/create_admin.py
   ```

3. **启动服务**
   ```bash
   # 启动 Bot
   python bot/main.py
   ```

4. **添加 Session 账号**
   - 通过 Web 界面或直接操作数据库添加 Session 账号
   - 确保至少有一个可用的 Session 账号
