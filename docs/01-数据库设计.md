# 数据库设计文档

## 概述

BlackHoleBot 使用 PostgreSQL 作为主数据库，Redis 作为缓存和临时存储。本文档详细描述所有数据表结构、索引设计和关系。

## 技术栈

- **数据库**: PostgreSQL 14+
- **ORM**: SQLAlchemy 2.0+
- **缓存**: Redis 7+

## 数据表设计

### 1. users - 用户表

存储所有使用 Bot 的用户信息。

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    telegram_id BIGINT UNIQUE NOT NULL,
    username VARCHAR(255),
    first_name VARCHAR(255),
    last_name VARCHAR(255),
    role VARCHAR(20) NOT NULL DEFAULT 'user',
    is_banned BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_active_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_telegram_id ON users(telegram_id);
CREATE INDEX idx_users_role ON users(role);
```

**字段说明**:
- `telegram_id`: Telegram 用户 ID（唯一标识）
- `role`: 用户角色
  - `user`: 普通用户
  - `vip`: VIP 用户（可访问 VIP 合集）
  - `admin`: 管理员
  - `super_admin`: 超级管理员
- `is_banned`: 是否被封禁

### 2. collections - 合集表

存储所有媒体合集信息。

```sql
CREATE TABLE collections (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    tags JSONB,
    deep_link_code VARCHAR(32) UNIQUE NOT NULL,
    access_level VARCHAR(20) NOT NULL DEFAULT 'public',
    media_count INTEGER DEFAULT 0,
    created_by INTEGER REFERENCES users(id) ON DELETE SET NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE UNIQUE INDEX idx_collections_deep_link ON collections(deep_link_code);
CREATE INDEX idx_collections_access_level ON collections(access_level);
CREATE INDEX idx_collections_name ON collections USING gin(to_tsvector('simple', name));
CREATE INDEX idx_collections_tags ON collections USING gin(tags);
```

**字段说明**:
- `deep_link_code`: 深链接唯一标识（如 `abc123`）
- `access_level`: 访问权限
  - `public`: 公开合集
  - `vip`: VIP 合集
- `tags`: 标签数组，使用 JSONB 存储，如 `["猫咪", "可爱"]`
- `media_count`: 媒体数量（冗余字段，便于统计）

### 3. media - 媒体文件表

存储合集中的所有媒体文件。

```sql
CREATE TABLE media (
    id SERIAL PRIMARY KEY,
    collection_id INTEGER NOT NULL REFERENCES collections(id) ON DELETE CASCADE,
    file_id VARCHAR(255) NOT NULL,
    file_unique_id VARCHAR(255) UNIQUE NOT NULL,
    file_type VARCHAR(20) NOT NULL,
    file_size BIGINT,
    caption TEXT,
    order_index INTEGER NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_media_collection_id ON media(collection_id);
CREATE INDEX idx_media_order ON media(collection_id, order_index);
CREATE UNIQUE INDEX idx_media_file_unique_id ON media(file_unique_id);
```

**字段说明**:
- `file_id`: Telegram 文件 ID（用于发送文件）
- `file_unique_id`: Telegram 唯一文件 ID（防止重复）
- `file_type`: 文件类型
  - `photo`: 图片
  - `video`: 视频
  - `document`: 文档
- `order_index`: 排序序号（用于分页展示）

### 4. session_accounts - Session 账号表

存储用于搬运任务的 Telegram 账号信息。

```sql
CREATE TABLE session_accounts (
    id SERIAL PRIMARY KEY,
    phone_number VARCHAR(20) UNIQUE NOT NULL,
    api_id INTEGER NOT NULL,
    api_hash VARCHAR(255) NOT NULL,
    session_string TEXT NOT NULL,
    priority INTEGER DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    transfer_count INTEGER DEFAULT 0,
    last_transfer_time TIMESTAMP,
    cooldown_until TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_used_at TIMESTAMP
);

CREATE INDEX idx_session_priority ON session_accounts(priority, is_active);
CREATE INDEX idx_session_cooldown ON session_accounts(cooldown_until);
```

**字段说明**:
- `session_string`: 加密后的 Telethon session 字符串
- `priority`: 优先级（数字越小越优先）
- `transfer_count`: 当前周期已转发数量（用于 500 限制）
- `cooldown_until`: 冷却结束时间

### 5. transfer_tasks - 搬运任务表

存储所有搬运任务信息。

```sql
CREATE TABLE transfer_tasks (
    id SERIAL PRIMARY KEY,
    task_name VARCHAR(255) NOT NULL,
    source_chat_id BIGINT NOT NULL,
    source_chat_username VARCHAR(255),
    filter_keywords JSONB,
    filter_type VARCHAR(20) DEFAULT 'all',
    filter_date_from TIMESTAMP,
    filter_date_to TIMESTAMP,
    session_account_id INTEGER REFERENCES session_accounts(id) ON DELETE SET NULL,
    status VARCHAR(20) NOT NULL DEFAULT 'pending',
    progress_current INTEGER DEFAULT 0,
    progress_total INTEGER DEFAULT 0,
    temp_redis_key VARCHAR(255),
    error_message TEXT,
    created_by INTEGER REFERENCES users(id) ON DELETE SET NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    started_at TIMESTAMP,
    completed_at TIMESTAMP
);

CREATE INDEX idx_tasks_status ON transfer_tasks(status);
CREATE INDEX idx_tasks_session ON transfer_tasks(session_account_id);
```

**字段说明**:
- `filter_keywords`: 过滤关键词数组，如 `["猫", "狗"]`
- `filter_type`: 过滤类型
  - `photo`: 仅图片
  - `video`: 仅视频
  - `all`: 全部
- `status`: 任务状态
  - `pending`: 等待执行
  - `running`: 执行中
  - `paused`: 已暂停
  - `waiting_bot`: 等待 Bot API 恢复
  - `completed`: 已完成
  - `failed`: 失败
- `temp_redis_key`: Redis 中临时存储的 key

### 6. task_logs - 任务日志表

记录搬运任务的详细日志。

```sql
CREATE TABLE task_logs (
    id SERIAL PRIMARY KEY,
    task_id INTEGER NOT NULL REFERENCES transfer_tasks(id) ON DELETE CASCADE,
    log_type VARCHAR(20) NOT NULL,
    message TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_logs_task_id ON task_logs(task_id, created_at);
```

**字段说明**:
- `log_type`: 日志类型
  - `info`: 普通信息
  - `warning`: 警告
  - `error`: 错误
  - `rate_limit`: 限流事件
  - `session_switch`: Session 切换

### 7. settings - 系统设置表

存储系统配置信息。

```sql
CREATE TABLE settings (
    key VARCHAR(255) PRIMARY KEY,
    value TEXT NOT NULL,
    description TEXT,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 初始化默认设置
INSERT INTO settings (key, value, description) VALUES
('welcome_message', '欢迎使用 BlackHoleBot！', '用户首次启动时的欢迎消息'),
('bot_name', 'BlackHoleBot', 'Bot 名称'),
('max_media_per_collection', '1000', '每个合集最大媒体数量');
```

**常用设置项**:
- `welcome_message`: 欢迎消息（支持 Telegram Premium Emoji）
- `bot_name`: Bot 名称
- `max_media_per_collection`: 每个合集最大媒体数量

## Redis 数据结构

### 1. 搬运任务临时文件存储

```
Key: task:{task_id}:files
Type: Set
Value: ["file_id_1", "file_id_2", ...]
TTL: 24 hours
```

### 2. 搬运任务待处理文件 ID

```
Key: task:{task_id}:pending
Type: Set
Value: [message_id_1, message_id_2, ...]
TTL: 1 hour
```

### 3. Bot 限流状态

```
Key: bot:rate_limited
Type: String
Value: "1" (限流中) / "0" (正常)
TTL: 10 minutes
```

### 4. 热门合集缓存

```
Key: collection:{deep_link_code}
Type: Hash
Fields: name, description, access_level, media_count
TTL: 1 hour
```

## 性能优化建议

### 1. 索引优化

- `collections.deep_link_code`: 唯一索引（高频查询）
- `collections.name`: GIN 全文索引（模糊搜索）
- `collections.tags`: GIN JSONB 索引（标签搜索）
- `media.collection_id + order_index`: 复合索引（分页查询）

### 2. 查询优化

**分页查询媒体**:
```sql
SELECT * FROM media
WHERE collection_id = ?
ORDER BY order_index
LIMIT 10 OFFSET ?;
```

**模糊搜索合集**:
```sql
SELECT * FROM collections
WHERE to_tsvector('simple', name) @@ plainto_tsquery('simple', ?)
   OR tags @> ?::jsonb
ORDER BY created_at DESC;
```

### 3. 定期清理

- 每天清理 30 天前的已完成任务日志
- 每周清理 Redis 过期的临时数据
- 每月归档旧的搬运任务记录

## 数据备份策略

1. **每日全量备份**: 使用 `pg_dump` 备份整个数据库
2. **实时 WAL 归档**: 启用 PostgreSQL WAL 归档
3. **Redis 持久化**: 启用 RDB + AOF 双重持久化

## 迁移脚本

使用 Alembic 进行数据库迁移管理。

```bash
# 创建迁移
alembic revision --autogenerate -m "Initial migration"

# 执行迁移
alembic upgrade head

# 回滚
alembic downgrade -1
```
